✅ Ejercicio 1

Tabla con números pares
Deseamos crear una tabla con una cantidad determinada de números pares. Para conseguirlo seguiremos los siguientes pasos:

  • Crear la tabla
  • Crear un procedimiento que inserte los n primeros números pares, es decir, 2, 4, 6, 8, …, 2 * n
  • Llamar al procedimiento y con una instrucción SELECT comprobar el resultado

Paso 1: Crear la tabla
Nombre de la tabla: NUMPARES
Estructura de la tabla:

Campo Tipo Tamaño Restricción
id_reg entero 5 clave primaria, auto incrementable
numero entero 11 no nulo

💡 Solución

-- Eliminar la tabla si existe
DROP TABLE IF EXISTS NUMPARES;

-- Crear la tabla
CREATE TABLE NUMPARES (
    id_reg INT(5) AUTO_INCREMENT,
    numero INT(11) NOT NULL,
    PRIMARY KEY (id_reg)
);

Paso 2: Crear procedimiento de relleno de la tabla
Creamos el procedimiento insertapares que reciba como parámetro la cantidad de números pares a insertar y los introduzca en la tabla.

💡 Solución

-- Eliminar el procedimiento si existe
DROP PROCEDURE IF EXISTS insertapares;

-- Creación del procedimiento
DELIMITER //
CREATE PROCEDURE insertapares(pNumReg INT)
BEGIN
    /* Declarar variables para el proceso y asignar valor inicial */
    DECLARE contador INT;
    DECLARE vNumero INT(11);
    SET vNumero := 0;
    SET contador := 1;

    WHILE ( contador <= pNumReg) DO
        /* Obtener valores para el insert */
        SET vNumero := vNumero + 2;
    
        /* Realizar el INSERT */
        INSERT INTO numpares (numero) VALUES (vNumero);
    
        /* Continuar con el bucle */
        SET contador := contador + 1;
    END WHILE;
END //
DELIMITER ;

Paso 3: Comprobar resultados
Llamar al procedimiento y comprobar que los resultados almacenados en la tabla son coherentes.

💡 Solución

-- Llamar al procedimiento (rellenar la tabla)
CALL insertapares(10);

-- Comprobar resultados
SELECT * FROM numpares;

✅ Ejercicio 2

Tabla con muestras de estaciones meteorológicas
DDeseamos crear una tabla con una cantidad determinada de registros que representan muestras tomadas por estaciones meteorológicas.
Para conseguirlo seguiremos los siguientes pasos:

  • Crear la tabla de muestras
  • Crear un procedimiento insertamuestras que rellene la tabla con n registros obtenidos con datos aleatorios
  • Llamar al procedimiento y ejecutar la instrucción SELECT para comprobar el resultado

Paso 1: Crear la tabla
Nombre de la tabla: MUESTRAS
Estructura de la tabla:

Campo Tipo Tamaño Restricción
id_reg entero 5 clave primaria, auto incrementable
ciudad cadena de carácters 50 no nulo
fecha fecha no nulo
hora tiempo no nulo
valida entero 1 valores: 0 y 1, defecto 1, no nulo
toma enumerado valores: ordinaria y extraordinaria; no nulo
temperatura decimal 5 con 1 decimal no nulo
humedad entero 3 no nulo
viento decimal 6 con 2 decimales no nulo
fecha_act marca de tiempo actualiza automáticamente con la fecha y hora del sistema, no nulo

💡 Solución

-- Eliminar la tabla si existe
DROP TABLE IF EXISTS MUESTRAS;

-- Crear la tabla
CREATE TABLE MUESTRAS (
    id_reg      INT(5) AUTO_INCREMENT,
    ciudad      VARCHAR(50) NOT NULL,
    fecha       DATE NOT NULL,
    hora        TIME NOT NULL,
    valida      INT(1) NOT NULL DEFAULT 1,
    toma        ENUM('ordinaria', 'extraordinaria') NOT NULL,
    temperatura DECIMAL(5,1) NOT NULL,
    humedad     INT(3) NOT NULL,
    viento      DECIMAL(6,2) NOT NULL,
    fecha_act   TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    PRIMARY KEY (id_reg)
);

Paso 2: Crear procedimiento de relleno de la tabla
Creamos el procedimiento insertamuestras que reciba como parámetro la cantidad de registros a insertar y los introduzca en la tabla con valores generados aleatoriamente. Para cada muestra generada deberemos obtener los siguientes datos:

Dato Descripción
Ciudad Obtendremos un número entero aleatorio de 3 posibles valores (0, 1 y 2) y dependiendo del resultado grabaremos ('Alicante', 'Valencia' o 'Castellón').
Fecha Obtendremos un número entero aleatorio entre 0 y 999 y le restaremos a la fecha del sistema tantos días como el número obtenido.
Hora Obtendremos un valor aleatorio para hora, minutos y segundos, y con MAKETIME crearemos el valor.
Toma Obtendremos un número entero aleatorio de 2 posibles valores (0 y 1) y dependiendo del resultado grabaremos ('ordinaria' o 'extraordinaria')
Valida Obtendremos un número entero aleatorio de 2 posibles valores (0 y 1) que grabaremos, pero deseamos que la probabilidad de obtener 1 sea del 90% y la de obtener 0 del 10%.
Temperatura Obtener una valor decimal entre -10º y -40º con una precisión de 1 decimal. Por ejemplo 15.3º.
Humedad Obtener un valor entero para el porcentaje de humedad entre 0 y 99.
Viento Obtener un valor decimal para la velocidad del viento en Km/h entre 0 y 149.99 con 2 decimales de precisión.

Los datos generados para cada muestra constituyen un registro que se tiene que insertar en la tabla.

💡 Solución

-- Eliminar el procedimiento si existe
DROP PROCEDURE IF EXISTS insertamuestras;

-- Creación del procedimiento
DELIMITER //
CREATE PROCEDURE insertamuestras(pNumReg INT)
BEGIN
    -- DELARACIÓN DE VARIABLES
    DECLARE contador INT;
    DECLARE numaleatorio INT;
    DECLARE vciudad VARCHAR(50);
    DECLARE vfecha DATE;
    DECLARE vhora TIME;
    DECLARE randhoras INT;
    DECLARE randminutos INT;
    DECLARE randsegundos INT;
    DECLARE vtoma VARCHAR(100);
    DECLARE vvalida tinyint(1);
    DECLARE vtemperatura DECIMAL(5,1);
    DECLARE vhumedad INT;
    DECLARE vviento DECIMAL(6,2);
    
    SET contador := 0;
    WHILE (contador < pNumReg) DO
        -- CIUDAD
        SET numaleatorio := FLOOR(3 * RAND());
        CASE numaleatorio
            WHEN 0 THEN
            SET vciudad := 'Alicante';
            WHEN 1 THEN
            SET vciudad := 'Valencia';
            WHEN 2 THEN
            SET vciudad := 'Castellón';
        END CASE;
        -- FECHA
        SET numaleatorio := FLOOR(1000 * RAND());
        SET vfecha := ADDDATE(CURDATE(), -numaleatorio);
        -- HORA
        SET randhoras := FLOOR(24 * RAND());
        SET randminutos := FLOOR(60 * RAND());
        SET randsegundos := FLOOR(60 * RAND());
        SET vhora := MAKETIME(randhoras, randminutos, randsegundos);
        -- TOMA
        SET numaleatorio := FLOOR(2 * RAND());
        IF (numaleatorio = 0) THEN
            SET vtoma := 'ordinaria';
        ELSE
            SET vtoma := 'extraordinaria';
        END IF;
        -- VALIDA con probabilidad del 90%
        SET numaleatorio := FLOOR(10 * RAND());
        IF (numaleatorio = 0) THEN
            SET vvalida := 0;
        ELSE
            SET vvalida := 1;
        END IF;
        -- TEMPERATURA ENTRE -10ºC y 40ºC
        SET numaleatorio := FLOOR(501 * RAND()) + (-100);
        SET vtemperatura := numaleatorio / 10;
        -- HUMEDAD en porcentaje de 0% a 99%
        SET numaleatorio := FLOOR(100 * RAND());
        SET vhumedad := numaleatorio;
        -- VIENTO ENTRE 0 Km/h y 149.99 Km/h con dos decimales
        SET numaleatorio := FLOOR(15000 * RAND());
        SET vviento := numaleatorio / 100;
        -- Insertar registro(id_reg y fecha_act toman valor automáticamente)
        INSERT INTO muestras (ciudad, fecha, hora, toma, valida, temperatura, humedad, viento)
        VALUES (vciudad, vfecha, vhora, vtoma, vvalida, vtemperatura, vhumedad, vviento);
        -- Continuar con el bucle
        SET contador := contador + 1;
    END WHILE;
END //
DELIMITER ;

Paso 3: Comprobar resultados
Llamar al procedimiento y comprobar que los resultados almacenados en la tabla son coherentes.

💡 Solución

-- Llamar al procedimiento (rellenar la tabla)
CALL insertamuestras(10);
