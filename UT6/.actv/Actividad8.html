Disparadores (Triggers)

Para la realización de estos ejercicios crearemos previamente una base de datos llamada testTriggers sobre la que realizaremos los ejercicios de la actividad.

✅ Ejercicio 1

Trigger INSERT en tabla de artículos

Este ejercicio consta de 3 pasos:

  • Creación de una tabla de artículos disponibles en un hipotético negocio.
  • Creación de un procedimiento que evite que se den de alta artículos con precio negativo.
  • Comprobación del funcionamiento de dicho procedimeiento.

Paso 1: Crear la tabla

Crear una tabla de nombre ARTICULOS con las siguientes características:

Campo Tipo Tamaño Restricción
codigo cadena de carácteres 10 clave primaria
descripcion cadena de carácteres 100 no nulo
precio decimal 8 con 2 decimales no nulo

Paso 2: Crear trigger
Crear un trigger de nombre insertArticulo que no permita que se inserte un artículo con un precio negativo.

Paso 3: Comprobación del funcionamiento
Para cada bloque de senténcias SQL interpreta los resultados.

-- Primer bloque
INSERT INTO ARTICULOS(codigo, descripcion, precio)
VALUES ('CAM01', 'Cámara de fotos PANASONIC FZ82', 265.88);
SELECT * FROM ARTICULOS;

-- Segundo bloque
INSERT INTO ARTICULOS(codigo, descripcion, precio)
VALUES ('CAM02', 'Cámara de fotos CANON Powershot Sx50', -322.00);
SELECT * FROM ARTICULOS;

✅ Ejercicio 2

Trigger UPDATE en tabla de artículos

Este ejercicio consta de 2 pasos:

  • Creación de un procedimiento que evite que se actualize el precio de un artículo con un valor negativo.
  • Comprobación del funcionamiento de dicho procedimeiento.

Paso 1: Crear trigger
Crear un trigger de nombre updateArticulo que no permita que se modifique el precio de un artículo con un valor negativo.

Paso 2: Comprobación del funcionamiento
Para cada bloque de senténcias SQL interpreta los resultados.

-- Primer bloque
UPDATE ARTICULOS SET precio = 280.15 WHERE codigo = 'CAM01';
SELECT * FROM ARTICULOS;

-- Segundo bloque
UPDATE ARTICULOS SET precio = precio - 300 WHERE codigo = 'CAM01';
SELECT * FROM ARTICULOS;

✅ Ejercicio 3

Trigger DELETE facturas

Este ejercicio consta de 3 pasos:

  • Creación de una tabla de facturas y otra con las líneas de facturas asociadas a las facturas.
  • Creación de un procedimiento que cuando se elimine una factura automáticamente elimine todas las líneas de factura de esa factura.
  • Comprobación del funcionamiento de dicho procedimeiento.

Paso 1: Crear las tablas

Crears las tablas FACTURAS y LINEASFACTURA con las siguientes características:

Tabla FACTURAS

Campo Tipo Tamaño Restricción
codigo cadena de carácteres 4 clave primaria
descripcion cadena de carácteres 100 no nulo
iva decimal 8 con 2 decimales no nulo

Tabla LINEASFACTURA

Campo Tipo Tamaño Restricción
codigo cadena de carácteres 4 clave ajena a FACTURAS
numero entero no nulo
articulo cadena de carácteres 10 clave ajena a ARTICULOS, no nulo
unidades entero 5 no nulo

La clave primaria está compuesta por codigo y numero.

Paso 2: Crear trigger
Crear un trigger de nombre deleteFactura que cuando se elimine una factura elimine todas sus líneas de factura asociadas.

Paso 3: Comprobación del funcionamiento
Para poder hacer pruebas tenemos que previamente insertar algunos valores en las tablas. Hecho esto, para cada bloque de sentencias SQL posterior interpreta los resultados.

-- Instrucciones preliminares
-- Insertar artículo 2
INSERT INTO ARTICULOS(codigo, descripcion, precio)
VALUES ('CAM02','Cámara de fotos CANON Powershot Sx50', 322.00);
-- Insertar facturas
INSERT INTO FACTURAS
VALUES ('A001', 'Factura A001: Balmis Shop Center', 21);
INSERT INTO FACTURAS
VALUES ('A002', 'Factura A002: Balmis Shop Center', 12);
-- Insertar líneas de factura de la factura
INSERT INTO LINEASFACTURA VALUES ('A001', 1,'CAM01', 2);
INSERT INTO LINEASFACTURA VALUES ('A001', 2,'CAM02', 4);
INSERT INTO LINEASFACTURA VALUES ('A002', 2,'CAM02', 6);

-- Primer bloque
SELECT * FROM FACTURAS;
SELECT * FROM LINEASFACTURA;

-- Segundo bloque
DELETE FROM FACTURAS WHERE codigo = 'A001'
SELECT * FROM FACTURAS;
SELECT * FROM LINEASFACTURA;
